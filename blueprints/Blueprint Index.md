# Blueprint Index

Use blueprints when you are actively solving a challenge. If you are still learning terminology, start at [Learning Path Index](../learning/Learning%20Path%20Index.md).

## Choose By Challenge Type

| Challenge type | Open first | Why |
|---|---|---|
| Unknown machine or boot2root | [Machine Exploitation Databank](Machine%20Exploitation%20Databank.md) | Fast picker by port, evidence, OS, and foothold signal. |
| Known machine attack type | [Machine Attack Blueprint Index](machine-attacks/Machine%20Attack%20Blueprint%20Index.md) | Direct templates for AD, SMB, web-to-shell, privesc, and service abuse. |
| Web-only challenge | [Web Exploit Blueprint](Web%20Exploit%20Blueprint.md) | Request workflow from recon to injection and source review. |
| Reversing/crackme/APK/WASM | [Reverse Engineering Blueprint](Reverse%20Engineering%20Blueprint.md) | Triage, static/dynamic analysis, and solver scripting. |
| Pwn/buffer overflow | [Buffer Overflow Blueprint](Buffer%20Overflow%20Blueprint.md) | Offset, mitigations, ret2win, ret2libc, ROP. |
| Steganography/file artifact | [Steganography Blueprint](Steganography%20Blueprint.md) | Metadata, embedded files, image/audio/text stego. |
| Crypto/encoding/hash | [Cryptography Blueprint](Cryptography%20Blueprint.md) | Encoding chains, XOR, RSA, hashes, custom crypto. |
