### 1. Initial Reconnaissance

Listing the files in `eos`'s home directory revealed an interesting file with a specific extension:

```bash
eos@hades:~$ ls -la
total 36
drwxr-x--- 2 root eos  4096 Apr  5  2024 .
drwxr-xr-x 1 root root 4096 Apr  5  2024 ..
...
-rw-r----- 1 root eos    22 Apr  5  2024 flagz.txt
-rw-r----- 1 root eos   181 Apr  5  2024 mission.txt
-r-xr-x--- 1 root eos  1902 Apr  5  2024 secretz.kbdx
```

#### Key Discovery:

The file `secretz.kbdx` is a **KeePass Password Database**. This aligns with the mission hint about Gaia being "careful saving her passwords."

### 2. File Identification and Transfer

To confirm the file type, the `file` command was used locally after transferring it via `scp`:

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ file secretz.kbdx
secretz.kbdx: Keepass password database 2.x KDBX
```

### 3. Cracking the Master Password

A KeePass database is encrypted and requires a master password to open. I used **John the Ripper** to crack it:

1. **Extract the hash**: Use `keepass2john` to convert the database into a format John can understand.
2. **Dictionary attack**: Use the `rockyou.txt` wordlist to find the master key.

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ keepass2john secretz.kbdx > hash

┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ john hash -w=/usr/share/wordlists/rockyou.txt
...
[REDACTED]           (secretz.kbdx)
...
```

The master password for the database is **`[REDACTED]`**.

### 6. Explanation

* **KDBX Files**: These are encrypted database files used by **KeePass**. They use strong encryption (like AES or ChaCha20), meaning they are highly secure unless the master password is weak enough to be guessed or cracked.
* **KeeWeb**: A useful cross-platform tool that allows you to open and inspect password databases directly in the browser without needing to install a dedicated desktop client.
* **Security Risk**: This challenge demonstrates that even if a user is "careful" enough to use a password manager, a weak master password allows an attacker to extract every secret stored within the vault once the file is obtained.
