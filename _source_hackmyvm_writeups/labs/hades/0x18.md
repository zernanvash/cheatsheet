In the home directory, I found a binary named `writeme` with both **SUID** and **SGID** bits set.

```bash
-r-s--s--- 1 root     calliope 16360 Apr  5  2024 writeme
```

### The Solution: TTY Messaging (mesg)

The binary is designed to "write" a message (the password) to your current terminal session. However, by default, many terminal environments have messaging disabled for security or privacy reasons.

#### Step 1: Initial Execution

When I first ran `./writeme`, the program repeatedly printed "Cannot send you my pass!". This indicated that the program was attempting to use the `write` or `wall` command logic but was being blocked by the current terminal settings.

```bash
calliope@hades:~$ ./writeme 
Cannot send you my pass!Cannot send you my pass!...
```

#### Step 2: Enabling Messages

In Linux, the `mesg` command controls whether other users (or privileged programs) can write to your terminal.

* `mesg n`: Disables messages (usually the default).
* `mesg y`: Enables messages.

I enabled terminal messaging to allow the privileged binary to send its data to my TTY.

```bash
calliope@hades:~$ mesg y
```

#### Step 3: Password Extraction

With messaging enabled, I ran the binary again. This time, the program successfully "wrote" the secret string into the middle of the output stream.

```bash
calliope@hades:~$ ./writeme
Cannot send you my pass!Cannot send you my pass!Cannot send you my pass![REDACTED]Cannot send you my pass!
```

### Explanation

* **SUID/SGID**: The `writeme` binary runs with elevated privileges, allowing it to access a password file owned by `root` or `calypso` that the user `calliope` cannot normally see.
* **The `write` Utility**: The standard `write` command allows a user to send a message to another user's terminal. If the recipient has `mesg n` set, the attempt is blocked.
* **The Finding**: The unique string leaked between the "Cannot send..." strings served as the password for user **calypso**.

### Execution

Using the retrieved password, I successfully established a clean SSH session as **calypso**.

```bash
calliope@hades:~$ ssh calypso@localhost
...
calypso@hades:~$ id
uid=2010(calypso) gid=2010(calypso) groups=2010(calypso)
```
