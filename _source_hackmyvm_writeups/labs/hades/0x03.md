# 0x03

moving from `alala` to `althea`

---

The mission for this stage provided a clear hint about user preferences:

```bash
alala@hades:~$ cat mission.txt
################
# MISSION 0x03 #
################

## EN ##
User althea loves reading Linux help.
```

In the home directory, I found a binary named `read` and a file named `althea_pass.txt` that I initially could not access due to permission restrictions.

```bash
alala@hades:~$ ls -la
total 52
...
-r--r----- 1 althea althea    21 Apr  5  2024 althea_pass.txt
-rw-r----- 1 root   alala     22 Apr  5  2024 flagz.txt
-rw-r----- 1 root   alala    164 Apr  5  2024 mission.txt
-rwS--s--- 1 root   alala  16056 Apr  5  2024 read
```

### The Solution:

The binary `read` had **SUID** (Set User ID) and **SGID** (Set Group ID) bits set. When executed, it displayed the effective user ID as `althea`. This indicated that the binary was designed to run with Althea's privileges.

```bash
alala@hades:~$ ./read
uid=2045(althea) gid=2044(alala) groups=2044(alala)
!done  (press RETURN)
[REDACTED]
!done  (press RETURN)
```

### Explanation:

* **SUID (Set User ID)**: The `S` in the owner's permission field (`-rwS--s---`) means this program runs with the authority of the file's owner (root) or a designated user .
* **SGID (Set Group ID)**: The `s` in the group's permission field allows the program to run with the privileges of the group `alala`.
* **Privilege Escalation**: In this specific challenge, the `read` binary was likely configured to read the `althea_pass.txt` file which was otherwise inaccessible to the `alala` user. By executing the binary, it used its elevated permissions to display the contents of the password file directly to the terminal.
* **The Findings**: The string `[REDACTED]` displayed by the program served as the cleartext password for user **althea**.

### Execution:

Using the identified password, I successfully logged in via SSH to establish a full session as `althea`.

```bash
alala@hades:~$ ssh althea@localhost
althea@hades:~$ id
uid=2045(althea) gid=2045(althea) groups=2045(althea)
```
