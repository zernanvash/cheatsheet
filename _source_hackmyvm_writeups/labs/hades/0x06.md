# 0x06

moving from `anthea` to `aphrodite`

---

The mission for this stage provided a very specific numerical clue:

```bash
anthea@hades:~$ cat mission.txt 
################
# MISSION 0x06 #
################

## EN ##
User aphrodite is obsessed with the number 94.
```

In the home directory, I found a binary named `obsessed` with **SUID** permissions, indicating it runs with elevated privileges.

```bash
anthea@hades:~$ ls -la
total 52
...
-r--r----- 1 aphrodite aphrodite    21 Apr  5  2024 aphrodite_pass.txt
-rwS--s--- 1 root      anthea    16256 Apr  5  2024 obsessed
```

### The Solution: Environment Variable Manipulation

When I first ran the program, it complained that a specific environment variable was missing: `No MYID ENV`. Based on the mission hint, I attempted to set this variable to **94**.

#### Step 1: Initialize the Environment Variable

I used the `export` command to create an environment variable named `MYID` with the value `94`.

```bash
anthea@hades:~$ export MYID=94
```

#### Step 2: Observe Program Behavior

Upon running the program again, it revealed that it was looking for a dynamic value. Each execution changed the required `MYID` value (e.g., from 57 to 53).

#### Step 3: Automate the Matching

To solve the "obsession," I realized the program was likely comparing the `MYID` environment variable against its own internal logic or a character representation. By setting `MYID` to the character `^` (which corresponds to the ASCII value 94), the program logic was satisfied.

```bash
anthea@hades:~$ export MYID=^
anthea@hades:~$ ./obsessed 
Current MYID: 94
aphrodite@hades:~$ id
uid=2048(aphrodite) gid=2047(anthea) groups=2047(anthea)
```

### Explanation

* **Environment Variables**: These are values used by the shell to pass information to programs. The `obsessed` binary was coded to check for a variable named `MYID`.
* **SUID Privilege Escalation**: Because the binary had the SUID bit set, once the "MYID" check passed, it spawned a shell with the privileges of the target user, **aphrodite**.
* **ASCII Obsession**: The number **94** is the decimal ASCII value for the caret symbol (`^`). By setting the environment variable to this character, the program's comparison logic (likely checking the byte value) succeeded.

### Execution

With the elevated shell, I retrieved the cleartext password for Aphrodite.

```bash
aphrodite@hades:~$ cat aphrodite_pass.txt 
[REDACTED]
```

Using the password, I successfully logged in via SSH to establish a clean session.

```bash
anthea@hades:~$ ssh aphrodite@localhost
aphrodite@hades:~$ id
uid=2048(aphrodite) gid=2048(aphrodite) groups=2048(aphrodite)
```
