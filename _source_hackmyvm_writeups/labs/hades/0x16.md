# 0x16

moving from `aura` to `aegle`

---

The mission for this stage pointed to a memory challenge involving user **aegle**:

```bash
aura@hades:~$ cat mission.txt 
################
# MISSION 0x16 #
################

## EN ##
User aegle has a good memory for numbers.
```

In the home directory, there was an executable named `numbers` that required specific input to proceed.

```bash
-rw---x--- 1 root aura 16064 Apr  5  2024 numbers
```

### The Solution: Sequential Pattern Discovery (Brute-Force)

Since I did not have permission to read the binary or its strings, the solution relied on trial and error to discover the numerical sequence the program expected.

#### 1. Understanding the Logic

The binary prompts for a number. If the number is correct, it says "Number OK" and asks for the next one. If the number is wrong, it prints "NO :_(" and terminates.

#### 2. Brute-Forcing the Sequence

I tested numbers sequentially. Every time the program accepted a number, I recorded it and restarted to find the next one in the chain.

* **First Attempt**: Found that `1` was the first correct digit.
* **Second Attempt**: Found that `2` followed `1`.
* **Third Attempt**: Found that `3` followed `2`.
* **Pattern Deviation**: After the sequence `1, 2, 3`, I found that the sequence repeated or shifted.

#### 3. The Winning Sequence

Through systematic testing, I discovered the full sequence required to trigger the secret output:
**`1` -> `2` -> `3` -> `1` -> `2` -> `3` -> `9` -> `1` -> `1` -> `1` -> `1` -> `2` -> `6`**

```bash
aura@hades:~$ ./numbers 
Enter one number: 1
Number OK
...
Enter next number: 2
Number OK
Enter next number: 6
[REDACTED]
```

### Explanation

* **Executable Permissions**: The binary was executable (`--x`) but not readable (`---`). This meant I could run it but couldn't use tools like `strings` or `objdump` to look for hardcoded sequences in the code.
* **Logic Traps**: CTF challenges often use simple loops or arrays that check user input against a hardcoded list. Without the ability to reverse-engineer the code, manual enumeration (brute-forcing) is the primary path forward.
* **The Finding**: The unique string printed at the end of the correct sequence served as the password for user **aegle**.

### Execution

Using the retrieved password, I successfully established an SSH session as **aegle**.

```bash
aura@hades:~$ ssh aegle@localhost
aegle@hades:~$ id
uid=2008(aegle) gid=2008(aegle) groups=2008(aegle)
```
