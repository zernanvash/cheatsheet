# 0x32

moving from `hebe` to `hera`

---

The mission for this stage focused on user **hera** and her preference for a communication service that is "older and open source" compared to Discord:

```bash
hebe@hades:~$ cat mission.txt 
################
# MISSION 0x32 #
################

## EN ##
User hera refuses to use Discord, she prefer an older and open source service.

## ES ##
La usuaria hera se niega a usar Discord, prefiere un medio mas antiguo y abierto.
```

### 1. Initial Reconnaissance

I began by checking for open ports on the local machine. Since the standard `netstat` command was missing, I used the `busybox` binary found in `/var/tmp`:

```bash
hebe@hades:~$ /var/tmp/busybox netstat -tlnup
Active Internet connections (only servers)
Proto Recv-Q Send-Q Local Address           Foreign Address         State       PID/Program name
...
tcp        0      0 127.0.0.1:6667          0.0.0.0:* LISTEN      -
...
```

#### Key Discovery:

Port **6667** is open on `localhost`. This is the standard port for **IRC (Internet Relay Chat)**, which perfectly fits the "older and open source" hint.

### 2. Identifying the Service

Running `netstat` with the `-a` flag showed that the system explicitly identifies this port as **ircd**:

```bash
hebe@hades:~$ /var/tmp/busybox netstat -atlup
...
tcp        0      0 localhost:ircd          0.0.0.0:* LISTEN      -
...
```

### 3. Interacting with the IRC Daemon

To retrieve the password, I needed to connect to the IRC server. Since no IRC client was installed, I used `busybox nc` (netcat).

IRC requires a registration handshake (NICK and USER) before it will accept commands. My first attempt timed out, so I quickly provided the necessary credentials on the second try:

```bash
hebe@hades:~$ /var/tmp/busybox nc localhost 6667
:hades.hmv NOTICE * :*** Looking up your hostname...
USER [REDACTED] 0 * [REDACTED]
NICK [REDACTED]
:hades.hmv 001 [REDACTED] :Welcome to the Devilnet IRC Network [REDACTED]!
...
:hades.hmv 376 [REDACTED] :End of message of the day.
```

### 4. Retrieving the Password

Once connected and registered, I used the `LIST` command to see all available channels and their topics:

```bash
LIST
:hades.hmv 321 [REDACTED] Channel :Users Name
:hades.hmv 322 [REDACTED] #channel666 0 :[+Pnt] Welcome hacker! Take it: [REDACTED]
:hades.hmv 323 [REDACTED] :End of channel list.
```

#### Discovery:

The password for **hera** was found in the topic of the `#channel666` channel: `[REDACTED]`

### 5. Execution

I used the discovered password to successfully log in as **hera** via SSH:

```bash
hebe@hades:~$ ssh hera@localhost
...
hera@hades:~$ id
uid=2024(hera) gid=2024(hera) groups=2024(hera)
```

---

### 6. Explanation

* **IRC (Internet Relay Chat)**: An application layer protocol that facilitates communication in the form of text. It is one of the oldest chat protocols still in use.
* **Manual Protocol Handshake**: Because IRC is a plain-text protocol, it is possible to interact with it using a simple TCP connection (`nc`). Providing the `NICK` (nickname) and `USER` (username/realname) is the standard way to register a session.
* **Information in Metadata**: In this level, the "flag" or credential was not stored in a file but rather in the **Channel Topic**, a piece of metadata provided by the IRC server when listing available chat rooms.
* **Local Service Binding**: The service was only listening on `127.0.0.1` (localhost), meaning it could only be accessed after gaining a shell as a local user.
