# 0x22

moving from `clio` to `cybele`

---

The mission for this stage focused on user **cybele** and her password choice:

```bash
clio@hades:~$ cat mission.txt 
################
# MISSION 0x22 #
################

## EN ##
The user cybele uses her lastname as a password.

## ES ##
La usuaria cybele usa su apellido como password.
```

### 1. Initial Reconnaissance

First, I checked for sudo privileges and searched for any files related to cybele:

```bash
clio@hades:~$ sudo -l
[sudo] password for clio: 
Sorry, user clio may not run sudo on hades.

clio@hades:~$ find / -name "*cybele*" -type f 2>/dev/null
```

No sudo privileges were available, and no cybele-specific files were found through the search.

### 2. Examining the Password File

I examined `/etc/passwd` to gather information about the cybele user:

```bash
clio@hades:~$ cat /etc/passwd
...
cybele:x:2014:2014:[REDACTED]:/pwned/cybele:/bin/bash
...
```

#### Key Discovery

The `/etc/passwd` entry for cybele reveals unusual information in the **GECOS field** (the field typically used for user's full name and other descriptive information):

* **Username**: `cybele`
* **UID/GID**: `2014`
* **GECOS field**: `[REDACTED]`
* **Home directory**: `/pwned/cybele`
* **Shell**: `/bin/bash`

### 3. The Solution: GECOS Field as Lastname

The GECOS field commonly contains the user's full name. In this case, the string `[REDACTED]` appears to be cybele's lastname, which according to the mission hint, she uses as her password.

### 4. Execution

Using the discovered lastname as the password, I successfully established an SSH session as **cybele**:

```bash
clio@hades:~$ ssh cybele@localhost
...
cybele@hades:~$ id
uid=2014(cybele) gid=2014(cybele) groups=2014(cybele)
```

### 5. Explanation

* **GECOS Field**: The GECOS field in `/etc/passwd` is the fifth field and traditionally stores user information such as full name, office number, and contact details. In this challenge, it was repurposed to store cybele's lastname.
* **Password Reuse**: The mission explicitly stated that cybele uses her lastname as her password, making this a straightforward credential discovery challenge once the lastname was identified.
* **Information Disclosure**: This level demonstrates how sensitive information inadvertently stored in world-readable system files (like `/etc/passwd`) can lead to account compromise.
