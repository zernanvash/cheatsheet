# 0x20

moving from `cassandra` to `cassiopeia`

---

The mission for this level focused on detecting "invisible" data left by user **cassiopeia**:

```bash
cassandra@hades:~$ cat mission.txt 
################
# MISSION 0x20 #
################

## EN ##
User cassiopeia sees the invisible. 
```

In the home directory, I found a file named `here.txt` containing a Base64 encoded string.

### The Solution: Steganography (Whitespace Analysis)

The hint about "seeing the invisible" combined with a file that appears empty or contains only whitespace after decoding points to **Whitespace Steganography**.

#### Step 1: Initial Decoding

I decoded the Base64 content to see the initial message. While it provided a prefix, the actual password appeared "invisible" (empty space).

```bash
cassandra@hades:~$ cat here.txt | base64 -d
The password of cassiopeia is:                               





cassandra@hades:~$
```

#### Step 2: Transferring the File

Since analyzing steganography often requires specific tools, I transferred `here.txt` to my local machine for further processing.

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ scp -P 6666 cassandra@hades.hackmyvm.eu:~/here.txt .
```

#### Step 3: Installing and Using Stegsnow

I identified that the "invisible" data was likely hidden using **Stegsnow**, a tool that hides messages in text files by appending tabs and spaces to the end of lines.

1. **Installation**: I installed the tool on my local Kali Linux environment.
```bash
sudo apt install stegsnow
```


2. **Extraction**: I piped the decoded Base64 content directly into `stegsnow` to reveal the hidden password.
```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ cat here.txt | base64 -d | stegsnow
[REDACTED]
```

### Explanation

* **Base64 Encoding**: This was the first layer of obfuscation used to transport the text and its hidden whitespace characters safely.
* **Whitespace Steganography**: This technique hides data in plain sight by using different combinations of spaces and tabs at the end of lines. Because these characters are usually not rendered by terminals or text editors, they are effectively "invisible."
* **Stegsnow**: This specialized utility specifically looks for these trailing whitespace patterns to reconstruct the original hidden message.
* **The Finding**: The hidden string `[REDACTED]` revealed by `stegsnow` served as the login password for **cassiopeia**.
