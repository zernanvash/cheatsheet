# 0x28

moving from `echo` to `eos`

---

The mission for this stage focused on user **eos** and the ability to "see" audio:

```bash
echo@hades:~$ cat mission.txt 
################
# MISSION 0x28 #
################

## EN ##
The user eos can see the sounds.

## ES ##
La usuaria eos puede ver los sonidos.
```

### 1. Initial Reconnaissance

After gaining access as `echo`, I listed the files in the home directory to identify potential vectors for moving to the next user:

```bash
echo@hades:~$ ls -la
total 468
drwxr-x--- 2 root echo   4096 Apr  5  2024 .
drwxr-xr-x 1 root root   4096 Apr  5  2024 ..
...
-rw-r----- 1 root echo     22 Apr  5  2024 flagz.txt
-rw-r----- 1 root echo    142 Apr  5  2024 mission.txt
-rw-r----- 1 root echo 442848 Dec 20  2021 noise.wav
```

#### Key Discovery:

There is an audio file named `noise.wav` that is readable by the `echo` group. Combined with the mission hint "eos can see the sounds," this strongly suggests **Audio Steganography**.

### 2. File Transfer and Identification

Since I cannot perform graphical audio analysis on the remote server, I transferred the file to my local machine using `scp`:

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ scp -P 6666 echo@hades.hackmyvm.eu:~/noise.wav .

```

I then verified the file type to ensure it was a standard audio format:

```bash
┌──(ouba㉿CLIENT-DESKTOP)-[~/hackmyvm]
└─$ file noise.wav
noise.wav: RIFF (little-endian) data, WAVE audio, Microsoft PCM, 16 bit, stereo 22050 Hz
```

### 3. Analyzing the Spectrogram

The hint "see the sounds" refers to a **Spectrogram**, which is a visual representation of the spectrum of frequencies in a sound as it varies with time.

I installed and used **Audacity** to analyze the file:

1. Open `noise.wav` in Audacity.
2. Click the three dots next to the track name (`noise`).
3. Select **Spectrogram** from the menu.

Upon switching to the spectrogram view, a text string became visible within the frequency patterns.

#### Discovery:

The hidden password revealed in the spectrogram.

### 4. Exploiting the Password

With the discovered string, I attempted to switch to the user **eos** via SSH:

```bash
echo@hades:~$ ssh eos@localhost
...
eos@hades:~$ id
uid=2020(eos) gid=2020(eos) groups=2020(eos)
```

### 5. Explanation

* **Audio Steganography**: This is the practice of hiding information within audio signals. In CTFs, this is often done by embedding text or images into the higher or lower frequency ranges of a WAV file.
* **Spectrogram Analysis**: While a waveform shows the amplitude of sound over time, a spectrogram shows the *frequency* over time. By carefully manipulating frequencies, creators can "draw" text that is invisible to the ear but clearly legible when visualized.
* **Mission Hint**: "See the sounds" is a common industry hint for tools like Audacity or Sonic Visualiser to inspect the spectral density of a file.
* **Pivot Strategy**: In this level, the privilege escalation was not based on a binary exploit (like the previous `ptx` level) but rather on finding hidden credentials left in a file accessible to your current group.
