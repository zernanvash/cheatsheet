moving from `asteria` to `astraea`

The mission clue for this level provided a cryptic hint regarding "magic":

```bash
asteria@hades:~$ cat mission.txt 
################
# MISSION 0x12 #
################

## EN ##
The user astraea believes in magic. 
```

A backup file found in the home directory, `sihiri_old.php`, revealed a PHP script using a vulnerable loose comparison for authentication:

```php
<?php
$pass = hash('md5', $_GET['pass']);
$pass2 = hash('md5',"ASTRAEA_PASS");
if($pass == $pass2){
print("ASTRAEA_PASS");
}
else{
print("Incorrect ^^");
}
?>
```

### The Solution: PHP Type Juggling (Magic Hashes)

The "magic" mentioned in the mission refers to **PHP Type Juggling**. When using the loose comparison operator (`==`), PHP treats strings that start with `0e` (followed only by numbers) as scientific notation (), which evaluates to **0**.

If both the user-supplied hash and the internal password hash start with `0e...`, PHP evaluates the condition as .

#### Step 1: Selecting a Magic Hash

Using a known list of MD5 magic hashes, I selected a value that produces a hash starting with `0e`:

* **Payload**: `240610708`
* **MD5 Hash**: `0e462097431906509019562988736854`

#### Step 2: Extracting the Password

By sending this payload to the active script via `curl`, the comparison logic was bypassed, and the script printed the cleartext password for the user.

```bash
asteria@hades:~$ curl localhost/sihiri.php?pass=240610708

[REDACTED]
```

### Explanation

* **Loose Comparison (`==`)**: This operator does not check for type identity. When it sees two strings that look like floats in scientific notation, it converts them both to , satisfying the `if` condition.
* **Magic MD5**: Since the string `240610708` produces a hash starting with `0e`, it successfully matched the internal secret hash, which also followed the "magic" `0e` pattern.
* **The Finding**: The cleartext password for the user **astraea** was successfully retrieved.
