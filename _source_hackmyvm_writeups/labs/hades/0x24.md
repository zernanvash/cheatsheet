### 2. Understanding the Clue: Gemini Protocol

The mission hint references "Gemini" and "gem-evil.hmv". **Gemini** is a lightweight internet protocol alternative to HTTP, typically running on port **1965**. The `.hmv` domain suggests this is an internal service on the target system.

### 3. Adding Local DNS Resolution

To access the service using the hostname `gem-evil.hmv`, I first added an entry to my local `/etc/hosts` file:

```bash
┌──(root㉿CLIENT-DESKTOP)-[/home/ouba]
└─# echo '127.0.0.1    gem-evil.hmv' >> /etc/hosts

┌──(root㉿CLIENT-DESKTOP)-[/home/ouba]
└─# cat /etc/hosts | grep 'gem'
127.0.0.1    gem-evil.hmv
```

### 4. Setting Up SSH Port Forwarding

To access the Gemini service running on the target's localhost, I established an SSH connection with **local port forwarding**:

```bash
┌──(root㉿CLIENT-DESKTOP)-[/home/ouba]
└─# ssh -p 6666 cynthia@hades.hackmyvm.eu -L 1965:127.0.0.1:1965
```

#### Explanation:
* **`-L 1965:127.0.0.1:1965`**: Forwards local port 1965 to port 1965 on the remote machine's localhost
* This allows me to access the Gemini server as if it were running locally

### 9. Explanation

* **Gemini Protocol**: A lightweight, privacy-focused alternative to HTTP that typically runs on port 1965. It uses a simple text-based format similar to Markdown.
* **SSH Port Forwarding**: The `-L` flag creates a secure tunnel that forwards traffic from a local port to a remote destination, allowing access to services bound to the remote machine's localhost.
* **Internal Services**: This challenge demonstrates how services running on localhost (127.0.0.1) are often hidden from external access and require either local access or port forwarding to reach.
* **Alternative Protocols**: The use of Gemini protocol highlights the importance of understanding lesser-known internet protocols during penetration testing and CTF challenges.
