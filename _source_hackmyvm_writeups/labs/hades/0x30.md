# 0x30

moving from `gaia` to `halcyon`

---

The mission for this stage focused on user **halcyon** and the concept of "powah" (power/privileges):

```bash
gaia@hades:~$ cat mission.txt
################
# MISSION 0x30 #
################

## EN ##
User halcyon wants all the powah.

## ES ##
La usuaria halcyon quiere todo el powah.
```

### 1. Initial Reconnaissance

Listing the files in `gaia`'s home directory revealed two files related to a password, but with different permissions:

```bash
gaia@hades:~$ ls -la 
...
-rw-r----- 1 root gaia    10 Apr  5  2024 hpass1.txt
-rw-r----- 1 root powah   23 Apr  5  2024 hpass2.txt
-rw-r----- 1 root gaia   146 Apr  5  2024 mission.txt
```

#### Key Discovery:

* `hpass1.txt` is readable by `gaia`.
* `hpass2.txt` is only readable by the group **powah**.
* `gaia` is currently NOT a member of the `powah` group.

### 2. Identifying the Target Group

I searched the system for other files belonging to the `powah` group to see if there were any hints on how to join it:

```bash
gaia@hades:~$ find / -group "powah" 2>/dev/null
/etc/w3m/ga
/pwned/gaia/hpass2.txt
```

#### Discovery:

There is a file at `/etc/w3m/ga` that also belongs to the `powah` group, but `hpass1.txt` in the home directory appeared more immediately relevant.

### 3. Escalating to the 'powah' Group

I read the first part of the password available to me:

```bash
gaia@hades:~$ cat hpass1.txt 
[REDACTED]
```

Since I needed the second part of the password stored in `hpass2.txt`, I attempted to switch my primary group to `powah` using the `newgrp` command. This command requires the group password if the user is not already a member.

```bash
gaia@hades:~$ newgrp powah
Password:
gaia@hades:~$
```

After successfully entering the password from `hpass1.txt`, my group ID changed:

```bash
gaia@hades:~$ id
uid=2021(gaia) gid=1000(powah) groups=1000(powah),2021(gaia)
```

### 4. Retrieving the Final Password

Now that I was part of the `powah` group, I could read the second half of the password:

```bash
gaia@hades:~$ cat hpass2.txt 
[REDACTED]
```

### 5. Execution

Using the password retrieved from `hpass2.txt`, I successfully logged in as **halcyon** via SSH:

```bash
gaia@hades:~$ ssh halcyon@localhost
...
halcyon@hades:~$ id
uid=2022(halcyon) gid=2022(halcyon) groups=2022(halcyon)
```

---

### 6. Explanation

* **Group Permissions**: In Linux, files can have specific permissions for a group. If you aren't in that group, you can't access the file even if it's in your own home directory (as seen with `hpass2.txt`).
* **newgrp Command**: This utility is used to change the current group ID during a login session. If a group has a password defined in `/etc/gshadow`, any user who knows that password can "log in" to that group even if they aren't listed as a member in `/etc/group`.
* **Information Splitting**: This challenge used a common CTF trope where a secret is split into two parts. You use the first part (the group password) to gain the privilege needed to see the second part (the user password).
