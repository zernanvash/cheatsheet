# 0x31

moving from `kira` to `veronica`

---

The mission said:

```bash
kira@venus:~$ cat mission.txt
################
# MISSION 0x31 #
################

## EN ##
The user veronica visits a lot http://localhost/waiting.php
```

I checked the home directory and identified the next target URL. Based on the previous levels, I knew I needed to interact with the local web server again.

```bash
kira@venus:~$ ls -la
total 32
...
-rw-r----- 1 root kira   31 Apr  5  2024 flagz.txt
-rw-r----- 1 root kira  191 Apr  5  2024 mission.txt
```

When I first accessed the page with a standard `curl` request, the server gave me a specific requirement:
`Im waiting for the user-agent PARADISE.`

To solve this, I needed to modify the **User-Agent** header in my HTTP request. The User-Agent string tells the server what kind of browser or tool is visiting the site.

### The Solution:

I attempted to use `-h`, but `curl` corrected me—the correct flag for custom headers is uppercase `-H`.

```bash
kira@venus:~$ curl -H "User-Agent: PARADISE" localhost/waiting.php
[REDACTED]
```

### Explanation:

* **`User-Agent`**: An HTTP header that identifies the client. Servers often use this to serve different content to mobile vs. desktop users, or in this case, to gate access to a secret.
* **`curl -H`**: The `-H` (Header) flag allows you to send any extra header you want.
* **The Logic**: The PHP script on the server likely checks if `$_SERVER['HTTP_USER_AGENT'] == 'PARADISE'`. If it matches, it prints the password.

With the password retrieved, I logged in as `veronica`.

```bash
kira@venus:~$ su - veronica
Password:

```
