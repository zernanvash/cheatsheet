# 0x26

moving from `alexa` to `ariel`

---

The mission said:

```bash
################
# MISSION 0x26 #
################

## EN ##
The password of the user ariel is online! (HTTP)
```

This hint indicates that the password is being hosted on a local web server. In Linux CTFs, "online" often means the password can be retrieved via a internal network request to the local machine (localhost).

To retrieve the password from the internal web service, I used the `curl` command. Since no port was specified, I assumed the default HTTP port (80).

```bash
alexa@venus:~$ curl localhost
[REDACTED]
```

### Explanation:

* **`curl`**: A command-line tool used to transfer data from or to a server. In this case, it acts like a web browser to fetch the content of a webpage.
* **`localhost`**: Refers to the local machine (IP `127.0.0.1`). This confirms a web server is running locally and serving the password string.

With the password obtained, I successfully switched to user `ariel`.

```bash
alexa@venus:~$ su - ariel
Password: 
ariel@venus:~$ id
uid=1027(ariel) gid=1027(ariel) groups=1027(ariel)
```
