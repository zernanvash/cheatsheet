# 0x28

moving from `lola` to `celeste`

---

The mission said:

```bash
lola@venus:~$ cat mission.txt
################
# MISSION 0x28 #
################

## EN ##
The user celeste has left a list of names of possible .html pages where to find her password.
```

Checking the home directory, I found a file named `pages.txt` containing a long list of potential filenames.

```bash
lola@venus:~$ ls -la
total 36
...
-rw-r----- 1 root lola  272 Apr  5  2024 mission.txt
-rw-r----- 1 root lola 1438 Apr  5  2024 pages.txt
```

Based on the mission, these pages are hosted on the local web server. I need to check each entry in `pages.txt` by appending `.html` and requesting it via `curl`. To automate this, I used a bash one-liner that loops through the list and stops only when it finds a page that actually exists.

```bash
lola@venus:~$ for page in $(cat pages.txt); do curl -s -f localhost/$page.html && echo -e "\nFound in: $page.html" && break; done
[REDACTED]

Found in: cebolla.html
```

### Explanation:

* **`for page in $(cat pages.txt);`**: Reads every word from the dictionary file.
* **`curl -s -f localhost/$page.html`**:
* `-s` (silent): Hides the progress meter.
* `-f` (fail): Ensures the command returns an error if the page is missing (404), allowing the `&&` logic to work.
* It builds the URL by adding the `.html` extension to each word.


* **`&& echo -e "\nFound in: $page.html" && break`**: If the request is successful, it prints the content (the password), identifies the source file (`cebolla.html`), and exits the loop.

With the password captured, I successfully logged in as `celeste`.
