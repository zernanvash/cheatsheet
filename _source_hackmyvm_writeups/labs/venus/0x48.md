# 0x48

moving from `belen` to `leona`

---

The mission clue indicated a clear case of credential theft:

```bash
belen@venus:~$ cat mission.txt 
################
# MISSION 0x48 #
################

## EN ##        
It seems that belen has stolen the password of the user leona...
```

I found a file named `stolen.txt` in the home directory. Checking its content revealed a **Unix hash string** rather than a cleartext password.

```bash
belen@venus:~$ cat stolen.txt 
$1$leona$lhWp56YnWAMz6z32Bw53L0
```

### The Strategy:

Since password cracking tools like `john` or `hashcat` were not installed on the remote server, I had to exfiltrate the hash to my local attack machine.

1. **Exfiltrate the Hash**: I used `scp` from my local desktop to download the file from the target server.
```bash
scp -P 5000 belen@venus.hackmyvm.eu:~/stolen.txt .
```

2. **Identify the Hash Type**: The `$1$` prefix identifies this as **MD5crypt** (MD5-based Unix password hash). In `hashcat`, this corresponds to mode **500**.
3. **Perform the Crack**: I ran `hashcat` using the famous `rockyou.txt` wordlist.
```bash
hashcat -m 500 stolen.txt /usr/share/wordlists/rockyou.txt
```

### Explanation:

* **MD5crypt (`$1$`)**: An older but still common hashing algorithm used in Unix-like systems. It uses a "salt" (`leona` in this case) to prevent rainbow table attacks.
* **Hashcat Mode 500**: This specific mode tells Hashcat to use the MD5crypt algorithm.
* **Dictionary Attack**: By comparing the stolen hash against the hashes of millions of common passwords (from `rockyou.txt`), the tool can find a match if the password is weak.
* **The Findings**: The attack was successful, revealing that the password for user `leona`.

### Execution:

With the password cracked on my local machine, I returned to the remote terminal to switch users.

```bash
belen@venus:~$ su - leona
Password: 
leona@venus:~$ id
uid=1049(leona) gid=1049(leona) groups=1049(leona)
```
