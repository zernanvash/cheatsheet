# 0x23

moving from `lucia` to `isabel`.

---

The mission said:

```bash
lucia@venus:~$ cat mission.txt
################
# MISSION 0x23 #
################

## EN ##
The user isabel has left her password in a file in the /etc/xdg folder but she does not remember the name, however she has dict.txt that can help her to remember.
```

Checking the home directory, I find the dictionary file mentioned in the mission.

```bash
lucia@venus:~$ ls -la
total 36
...
-rw-r----- 1 root  lucia 1998 Apr  5  2024 dict.txt
-rw-r----- 1 root  lucia  397 Apr  5  2024 mission.txt
```

Since I don't know which word in `dict.txt` matches the filename in `/etc/xdg`, I can use a bash loop to test every word. I pipe the errors (`2>/dev/null`) so I only see the file that actually exists.

```bash
lucia@venus:~$ for word in $(cat dict.txt); do ls /etc/xdg/$word 2>/dev/null; done
/etc/xdg/readme
```

The loop found a match: `/etc/xdg/readme`. Now, I simply read that file to get the password.

```bash
lucia@venus:~$ cat /etc/xdg/readme
[REDACTED]
```

Login using the discovered password.

```bash
lucia@venus:~$ su - isabel

isabel@venus:~$ id
uid=1024(isabel) gid=1024(isabel) groups=1024(isabel)
```
