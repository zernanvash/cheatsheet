# 0x36

moving from `gloria` to `alora`

---

The mission said:

```bash
gloria@venus:~$ cat mission.txt
################
# MISSION 0x36 #
################

## EN ##
User alora likes drawings, that's why she saved her password as ...
```

I checked the home directory and found a file named `image`. Unlike previous levels where images were binary files (like `.png`), this one was identified as text.

```bash
gloria@venus:~$ ls -la
total 36
...
-rw-r----- 1 root   gloria 1713 Apr  5  2024 image
-rw-r----- 1 root   gloria  222 Apr  5  2024 mission.txt
gloria@venus:~$ file image
image: ASCII text
```

When I displayed the content of the file using `cat`, it revealed a **QR Code** rendered entirely in **ASCII art** using `#` characters and spaces.

### The Solution:

1. **Phone Scan**: Simply shrinking the terminal font size and scanning the screen with a smartphone QR reader.

### Explanation:

* **ASCII Art QR Code**: A creative way to store data in a text-only environment. Each `#` represents a dark module and each space represents a light module.
* **Scanning**: Most modern QR scanners are robust enough to recognize the pattern even when represented by text characters instead of pixels.
* **The Findings**: Scanning the ASCII "drawing" successfully decoded the string hidden within the modules.

The decoded QR code provided the password.

```bash
gloria@venus:~$ su - alora
Password: 
alora@venus:~$ id
uid=1037(alora) gid=1037(alora) groups=1037(alora)
```
