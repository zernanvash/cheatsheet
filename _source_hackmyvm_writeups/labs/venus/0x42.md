# 0x42

moving from `sarah` to `mercy`

---

The mission for this stage was straightforward but required a keen eye for directory contents:

```bash
sarah@venus:~$ cat mission.txt 
################
# MISSION 0x42 #
################

## EN ##        
The password of mercy is hidden in this directory.
```

I listed the files in the current directory and noticed an unusual entry that at first glance might look like a directory navigation shortcut (`.` or `..`), but was actually a file named `...`.

```bash
sarah@venus:~$ ls -la
total 36
drwxr-x--- 2 root  sarah 4096 Apr  5  2024 .  
drwxr-xr-x 1 root  root  4096 Apr  5  2024 .. 
-rw-r----- 1 root  sarah   16 Apr  5  2024 ...  <-- Hidden file
...
-rw-r----- 1 root  sarah   31 Apr  5  2024 flagz.txt   
-rw-r----- 1 root  sarah  175 Apr  5  2024 mission.txt 
```

### The Solution:

In Linux, any file starting with a dot (`.`) is hidden from a standard `ls` command. While we often see `.` (current directory) and `..` (parent directory), a file named `...` is a common CTF trick to hide data in plain sight by mimicking those system pointers.

```bash
sarah@venus:~$ cat ./...
[REDACTED]
```

### Explanation:

* **Hidden Files**: Files that begin with a `.` character are considered hidden and only appear when using the `-a` (all) flag with `ls`.
* **Path Specification**: When a filename is composed only of dots, it is best to use the full path `./...` to ensure the shell interprets it as a file in the current directory rather than a miskeyed navigation command.
* **The Findings**: The file `...` contained the cleartext password needed to advance.

Using the retrieved password, I successfully switched to user `mercy`.

```bash
sarah@venus:~$ su - mercy
Password: 
mercy@venus:~$ id
uid=1043(mercy) gid=1043(mercy) groups=1043(mercy)
```
