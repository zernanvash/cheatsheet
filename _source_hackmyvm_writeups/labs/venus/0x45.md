# 0x45

moving from `karla` to `denise`

---

The mission clue was centered on an image file:

```bash
karla@venus:~$ cat mission.txt 
################
# MISSION 0x45 #
################

## EN ##        
User denise has saved her password in the image.
```

In the home directory, I found a JPEG image named `yuju.jpg`.

```bash
karla@venus:~$ ls -la
total 68
...
-rw-r----- 1 root  karla   31 Apr  5  2024 flagz.txt  
-rw-r----- 1 root  karla  176 Apr  5  2024 mission.txt
-rw-r----- 1 root  karla 32946 Apr  5  2024 yuju.jpg   
```

### The Solution:

Since the password was reportedly "in the image," but not necessarily visible in the picture itself, I checked the file's **metadata**. Metadata contains hidden information about the file, such as the creator, the date it was taken, or custom comments. I used `exiftool` to inspect these hidden tags.

```bash
karla@venus:~$ exiftool yuju.jpg 
...
Artist                          : sML
Date/Time Original              : 2021:11:01 10:34:51
...
About                           : [REDACTED]
Creator                         : sML
...
```

### Explanation:

* **Metadata (EXIF/XMP)**: Digital images often store extra data in headers. While some tags like `Image Width` are standard, others like `About`, `Artist`, or `Comment` can be used to hide strings or passwords.
* **`exiftool`**: A powerful command-line application used for reading, writing, and editing meta-information in a wide variety of files.
* **The Finding**: Among the metadata fields, the `About` tag contained a suspicious 15-character string that did not look like standard metadata.

The string found in the `About` field served as the password for the next user.

Using the identified password, I successfully logged in as `denise`.

```bash
karla@venus:~$ su - denise
Password: 
denise@venus:~$ id
uid=1046(denise) gid=1046(denise) groups=1046(denise)
```
