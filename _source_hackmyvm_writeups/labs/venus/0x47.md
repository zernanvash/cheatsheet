moving from `zora` to `belen`

The mission for this level pointed toward a specific domain name:

```bash
zora@venus:~$ cat mission.txt

The user belen has left her password in venus.hmv
```

I checked the home directory and found a file named `zora_pass.txt`, but the hint specifically directed me toward an external or internal web resource named `venus.hmv`.

```bash
zora@venus:~$ ls -la
total 36

```

### The Solution:

The address `venus.hmv` suggests a Virtual Host or a local domain defined in the system's `/etc/hosts` file. To retrieve the password, I used `curl` to make an HTTP request to that domain.

```bash
zora@venus:~$ curl venus.hmv
[REDACTED]
```

### Explanation:

* **Local Domains**: On Linux systems, names like `venus.hmv` can be mapped to IP addresses (usually `127.0.0.1`) via the `/etc/hosts` file. This allows local web services to be accessed using friendly names instead of just `localhost`.
* **`curl`**: Since no port was specified, `curl` defaults to port 80 (HTTP). The web server configured for `venus.hmv` was set up to return the password string as the body of the response.
* **Virtual Hosting**: The server likely uses name-based virtual hosting, where it serves different content depending on the "Host" header provided in the request.

Using the retrieved password, I successfully logged in as `belen`.

