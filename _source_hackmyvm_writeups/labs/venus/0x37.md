# 0x37

moving from `alora` to `julie`

---

The mission for this stage was:

```bash
alora@venus:~$ cat mission.txt
################
# MISSION 0x37 #
################

## EN ##
The user julie has created an iso with her password.
```

In the home directory, I found a large file named `music.iso`. Verification with the `file` command confirmed it was an ISO 9660 filesystem image.

```bash
alora@venus:~$ file music.iso
music.iso: ISO 9660 CD-ROM filesystem data 'CDROM'
```

Initially, I tried to `mount` the image to view its contents, but since I lacked `sudo` privileges and did not have the `7z` tool installed to extract it, I needed a way to read the data within the binary file directly.

### The Solution:

Since an ISO is a raw image of a disk and the password was stored in a text file (`music.txt`) inside it, the text remained uncompressed and readable. I used the `strings` command to scan the binary file for printable character sequences.

```bash
alora@venus:~$ strings music.iso 
...
pwned/alora/music.txt
[REDACTED]
pwned/alora/music.txt
...
```

### Explanation:

* **`strings`**: This tool is used to find and display printable strings within a binary file. It is incredibly useful for analyzing non-text files like compiled binaries, disk images, or memory dumps.
* **ISO Structure**: In an uncompressed ISO 9660 image, the content of internal text files is often stored in plain ASCII or UTF-8. By running `strings`, I was able to see the internal path (`pwned/alora/music.txt`) and the actual password string immediately following it.
* **Bypassing Restrictions**: This method allowed me to find the secret without needing to mount the disk or possess administrative permissions.

Using the identified password, I successfully logged in as `julie`.

```bash
alora@venus:~$ su - julie
Password: 
julie@venus:~$ id
uid=1038(julie) gid=1038(julie) groups=1038(julie)
```
