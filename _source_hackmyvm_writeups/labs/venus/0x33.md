# 0x33

moving from `lana` to `noa`

---

The mission said:

```bash
lana@venus:~$ cat mission.txt
################
# MISSION 0x33 #
################

## EN ##
The user noa loves to compress her things.
```

I checked the home directory and found a file named `zip.gz`. Despite the double extension, the file type wasn't immediately clear.

```bash
lana@venus:~$ ls -la
total 44
...
-rw-r----- 1 root lana 10240 Apr  5  2024 zip.gz
```

Initially, I tried to use `gunzip`, but it failed. I then used the `file` command to determine the actual file type, which revealed it was a **POSIX tar archive**, not a gzip or zip file.

```bash
lana@venus:~$ file zip.gz
zip.gz: POSIX tar archive (GNU)
```

### The Solution:

Since I didn't have write permissions in the current folder, I created a temporary workspace in `/tmp/`, copied the archive there, and extracted it using `tar`.

```bash
lana@venus:~$ mkdir /tmp/laanaa/
lana@venus:~$ cp zip.gz /tmp/laanaa/
lana@venus:~$ cd /tmp/laanaa/
lana@venus:/tmp/laanaa$ tar -xf zip.gz
```

### Explanation:

* **`file`**: A vital tool in CTFs to verify the actual format of a file, as extensions can be misleading.
* **`mkdir /tmp/laanaa/`**: Creating a directory in `/tmp` is a common tactic when you need a writable area to extract files.
* **`tar -xf`**:
* `-x`: Extract the contents.
* `-f`: Specify the filename.


* **Extraction Path**: Navigating through the extracted folders (`pwned/lana/`) led to a plain text file named `zip`.

Inside the extracted folder structure, I found the password:

```bash
lana@venus:/tmp/laanaa/pwned/lana$ cat zip
[REDACTED]
```

Using the password, I successfully switched to user `noa`.

```bash
lana@venus:~$ su - noa
Password: 
noa@venus:~$ id
uid=1034(noa) gid=1034(noa) groups=1034(noa)
```
