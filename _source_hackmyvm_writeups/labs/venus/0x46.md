# 0x46

moving from `denise` to `zora`

---

The mission clue was centered on a specific binary utility:

```bash
denise@venus:~$ cat mission.txt 
################
# MISSION 0x46 #
################

## EN ##        
The user zora is screaming doas!
```

I checked the home directory and confirmed there were no obvious password files or images, suggesting the solution involved the system configuration mentioned in the hint.

```bash
denise@venus:~$ ls -la
total 32
...
-rw-r----- 1 root   denise   31 Apr  5  2024 flagz.txt  
-rw-r----- 1 root   denise  144 Apr  5  2024 mission.txt
```

### The Solution:

The clue "doas" refers to a utility (originally from OpenBSD) that allows a user to execute commands as another user, similar to `sudo`. Since `denise` was not allowed to use `sudo`, I tested the permissions for `doas`.

By executing `/bin/bash` through `doas` while specifying `zora` as the target user, I was able to spawn a shell with Zora's privileges using Denise's own password.

```bash
denise@venus:~$ doas -u zora /bin/bash
doas (denise@venus) password: 
zora@venus:/pwned/denise$ id
uid=1047(zora) gid=1047(zora) groups=1047(zora)
```

### Explanation:

* **`doas` (Dedicated OpenBSD Application Subexecutor)**: A lightweight alternative to `sudo`. It is configured via `/etc/doas.conf`.
* **Privilege Escalation**: In this configuration, the administrator granted `denise` the right to execute commands as `zora`.
* **The Command**:
* **`-u zora`**: Specifies that the command should be run as the user `zora`.
* **`/bin/bash`**: The command to execute. Running a shell directly is an easy way to move from one user to another without needing the target's password.


* **Authentication**: Unlike `su` (which requires the *target* user's password), `doas` usually requires the *current* user's password to verify identity before granting the requested permissions.

Once the shell was spawned, I successfully transitioned to user `zora`.

```bash
zora@venus:/pwned/denise$ cd
zora@venus:~$ id
uid=1047(zora) gid=1047(zora) groups=1047(zora)
```
