# 0x40

moving from `adela` to `sky`

---

The mission clue was quite literal:

```bash
adela@venus:~$ cat mission.txt 
################
# MISSION 0x40 #
################

## EN ##
User sky has saved her password to something that can be listened to.
```

In the home directory, the file `wtf` was identified as a simple ASCII text file.

```bash
adela@venus:~$ file wtf
wtf: ASCII text
```

Upon viewing the file, it revealed a sequence of dots and dashes, which is the standard representation for **Morse Code**.

```bash
adela@venus:~$ cat wtf
.--. .- .--. .- .--. .- .-. .- -.. .. ... .
```

### The Solution:

Since the local system lacked Python or specialized decoding tools, you used **CyberChef** to decode the string. By applying the "From Morse Code" operation with a "Space" letter delimiter, the sequence was successfully translated.

* **Input**: `.--. .- .--. .- .--. .- .-. .- -.. .. ... .`

### Explanation:

* **Morse Code**: A method of encoding text characters as standardized sequences of dots and dashes. It fits the mission hint perfectly as Morse was traditionally transmitted via audio signals.
* **CyberChef**: A powerful web-based "Swiss Army Knife" for encryption, encoding, and data analysis. It is ideal for decoding Morse code when terminal-based tools are unavailable.
* **Password Normalization**: While the decoded output was in uppercase, you correctly identified that the actual system password was **lowercase**.

With the password identified, the transition to user `sky` was successful.
