# 0x50

moving from `ava` to `maria`

---

The mission clue was intentionally vague, indicating the secret was "somewhere" other than a file in the current directory:

```bash
ava@venus:~$ cat mission.txt 
################
# MISSION 0x50 #
################

## EN ##
The password of maria is somewhere...
```

I checked the local directory, but as expected, no obvious password files were found.

```bash
ava@venus:~$ ls -la
total 32
...
-rw-r----- 1 root ava    31 Apr  5  2024 flagz.txt
-rw-r----- 1 root ava   153 Apr  5  2024 mission.txt
```

### The Solution:

The password was hidden within the profile data of a previously encountered user. By investigating the home directory of the user **sky**.

### Explanation:

* **Lateral Information Gathering**: In CTF challenges, it is common for a user's password to be stored in a location accessible to a previous user or hidden in a profile you have already compromised.
* **The "Somewhere" Clue**: This is a direct hint that you need to look outside the current working directory and revisit system locations or other user homes that you have permission to read.
* **The Finding**: The password was specifically located in the directory or files belonging to user **sky**, rather than in an environment variable or a local file in `ava`'s home.

Using the identified password, I successfully switched to user `maria`.

```bash
ava@venus:~$ su - maria
Password: 
maria@venus:~$ id
uid=1051(maria) gid=1051(maria) groups=1051(maria)
```
