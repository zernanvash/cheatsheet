Overview
--------
This crackme consists of two programs:
- note_writer: creates encrypted notes using a password.
- note_reader: decrypts notes using the password.

The goal was to understand the encryption and decrypt a note (flag.note) without knowing the password.

Analysis
--------
1. note_writer prompts the user for a password and note content.
2. The password is passed to FUN_00101340, which calculates a 32-bit key using the Adler-32 checksum algorithm:

   if length == 0:
       A = 1
       B = 0
   else:
       A = 1
       B = 0
       for each byte in password:
           A = (A + byte) % 0xFFF1
           B = (B + A) % 0xFFF1
   return A + B*0x10000

3. FUN_001013c0 performs the encryption by XORing each byte of the note with (i + key_bytes[i % 4]), where i is the byte index.
4. The resulting note is written to a file. File structure:

   Offset 0-3  : 32-bit key (Adler-32 of password)
   Offset 4-N  : Encrypted note content

Decryption
----------
Because XOR is reversible and the key is stored in the file, the original password is not needed. The decryption process is:

import struct

def decrypt_file(path):
    with open(path, "rb") as f:
        data = f.read()
    key = struct.unpack("<I", data[:4])[0]
    encrypted = data[4:]
    key_bytes = key.to_bytes(4, "little")
    return bytearray(b ^ ((i + key_bytes[i % 4]) & 0xFF) for i, b in enumerate(encrypted))

This script decrypts any .note file generated by note_writer.

Conclusion
----------
- The key is the Adler-32 checksum of the password and is stored in plaintext in the file. (not secure!)
- Encryption uses XOR with a 4-byte key plus the byte index.
- Decryption is trivial once the key is read from the file.
- Security is very weak: knowing the key allows decryption without the password.
