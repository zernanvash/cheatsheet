## Triage

The target is a small stripped Linux x86-64 PIE executable. It imports only a few libc functions relevant to the password check:

The visible strings include the usage text, the failure message, and the success message:

The interesting part is that there is no plaintext password string. The program generates the comparison string at runtime.

## Main Function

IDA decompiles `main` to the following behavior:

```c
int main(int argc, char **argv)
{
    char *input;
    char *generated;

    if (argc != 2) {
        printf("usage: %s <password>\n", argv[0]);
        exit(1);
    }

    input = argv[1];
    generated = sub_1220(input);

    if (strcmp(input, generated) == 0)
        printf("You cracked the password\nGreat job!!\n\a");
    else
        puts("Wrong password");

    free(generated);
    return 0;
}
```

So the success condition is:

```text
input == sub_1220(input)
```

This makes the challenge a fixed-point problem.

## Password Generation Function

The function at `0x1220` allocates a 26-byte buffer, writes 25 generated bytes plus a null terminator, and returns it.

It first copies hardcoded bytes from `.rodata`:

```text
0x2060: 79 47 6e 7d 6a 61 47 6b 6c
0x2070: 6a 77 76 7f 47 68 79 6b 6b 6f 77 6a 7c 2d 28 2f
```

Because the second 16-byte copy starts at output offset 9, the actual 25-byte seed is:

```text
79 47 6e 7d 6a 61 47 6b 6c 6a 77 76 7f 47 68 79 6b 6b 6f 77 6a 7c 2d 28 2f
```

In byte-string form:

```text
yGn}jaGkljwv.Ghykkowj|-(/    ; the dot represents byte 0x7f
```

Then the function computes the XOR of every input byte. Call that one-byte value `k`.

Finally, it XORs every seed byte with `k`:

```c
for (i = 0; i < 25; i++)
    generated[i] = seed[i] ^ k;
generated[25] = 0;
```

Therefore the required password must satisfy:

```text
password = seed ^ xor(password)
```

## Solving

Let:

```text
password[i] = seed[i] ^ k
```

For this to be valid, `k` must also equal the XOR of all password bytes:

```text
k = xor(seed[0] ^ k, seed[1] ^ k, ..., seed[24] ^ k)
```

There are 25 bytes, an odd count, so XORing `k` 25 times leaves one `k`:

```text
xor(password) = xor(seed) ^ k
```

The seed bytes XOR to zero:

```text
xor(seed) = 0x00
```

So any one-byte `k` gives a mathematical fixed point:

```text
xor(password) = 0x00 ^ k = k
```

The intended printable password appears when `k = 0x18`:

```text
seed ^ 0x18 = a_very_strong_password507
```

Small Python verification:

```python
seed = bytes.fromhex(
    "79476e7d6a61476b6c"
    "6a77767f4768796b6b6f776a7c2d282f"
)

password = b"a_very_strong_password507"
k = 0
for b in password:
    k ^= b

generated = bytes(b ^ k for b in seed)

print(hex(k))
print(generated)
print(generated == password)
```

Output:

```text

b'a_very_strong_password507'
True
```
